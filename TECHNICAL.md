# BeeWatch Technical Documentation

## Architecture Overview

BeeWatch is a watchOS app with an iOS companion, built using SwiftUI and targeting iOS 17+ / watchOS 10+. The app allows users to track and enter data for their Beeminder goals directly from their Apple Watch.

## Core Components

### Models

#### Goal (`Shared/Models/Goal.swift`)

SwiftData model representing a Beeminder goal.

```swift
@Model
final class Goal: Identifiable {
    @Attribute(.unique) var slug: String
    var title: String
    var safebuf: Int          // Days of safety buffer
    var losedate: Date        // Derailment deadline
    var rate: Double          // Goal rate
    var pledge: Double        // Money at risk
    // ... other fields

    var urgencyColor: UrgencyLevel  // Computed: red/orange/blue/green
    var timeRemaining: String       // Computed: "2d 5h" format
}
```

**Urgency Levels:**
- **Red**: `safebuf <= 0` (derailing today)
- **Orange**: `safebuf == 1` (derailing tomorrow)
- **Blue**: `safebuf <= 3` (due soon)
- **Green**: `safebuf > 3` (safe)

#### Datapoint (`Shared/Models/Datapoint.swift`)

```swift
@Model
final class Datapoint: Identifiable {
    @Attribute(.unique) var datapointId: String
    var goalSlug: String
    var timestamp: Date
    var value: Double
    var comment: String
}
```

#### UserSettings (`Shared/Models/UserSettings.swift`)

Singleton managing API credentials with Keychain storage.

```swift
@Observable
final class UserSettings {
    static let shared = UserSettings()

    var apiKey: String      // Stored in Keychain
    var username: String    // Stored in UserDefaults
    var isConfigured: Bool  // Computed from apiKey
}
```

**Storage:**
- API Key: Stored in iOS Keychain with `kSecAttrAccessibleAfterFirstUnlock`
- Username: Stored in UserDefaults

### Services

#### BeeminderAPI (`Shared/Services/BeeminderAPI.swift`)

Actor-based API client using async/await.

```swift
actor BeeminderAPI {
    static let shared = BeeminderAPI()

    // Fetch all goals (sorted by urgency)
    func fetchGoals() async throws -> [GoalResponse]

    // Fetch datapoints for a goal
    func fetchDatapoints(goalSlug: String, count: Int) async throws -> [DatapointResponse]

    // Create a new datapoint
    func createDatapoint(goalSlug: String, value: Double, comment: String) async throws -> DatapointResponse
}
```

**API Endpoints:**
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/users/{user}/goals.json` | List all goals |
| GET | `/users/{user}/goals/{slug}.json` | Get single goal |
| GET | `/users/{user}/goals/{slug}/datapoints.json` | List datapoints |
| POST | `/users/{user}/goals/{slug}/datapoints.json` | Create datapoint |

**Authentication:** All requests include `auth_token` parameter.

**Error Handling:**
```swift
enum APIError: LocalizedError {
    case notConfigured      // No API key
    case invalidURL
    case invalidResponse
    case httpError(Int, String?)
    case decodingError(Error)
    case networkError(Error)
}
```

#### DataStore (`Shared/Services/DataStore.swift`)

Main state container using `@Observable` pattern.

```swift
@MainActor
final class DataStore: ObservableObject {
    static let shared = DataStore()

    @Published var goals: [Goal] = []
    @Published var isLoading = false
    @Published var error: Error?

    func refreshGoals() async
    func fetchDatapoints(for goalSlug: String) async throws -> [Datapoint]
    func submitDatapoint(goalSlug: String, value: Double, comment: String) async throws -> Datapoint

    // Frequent values tracking
    func getFrequentValues(for goalSlug: String, limit: Int) -> [FrequentValue]
}
```

**Frequent Values:** Tracks commonly entered values per goal for quick entry buttons.

#### NotificationManager (`Shared/Services/NotificationManager.swift`)

Handles reminder scheduling using UserNotifications framework.

```swift
@MainActor
final class NotificationManager: ObservableObject {
    static let shared = NotificationManager()

    func requestAuthorization() async -> Bool
    func scheduleReminder(for setting: ReminderSetting) async
    func scheduleUrgentReminder(for goal: Goal) async
    func setupNotificationCategories()  // Quick-entry actions
}
```

**Notification Categories:**
- `GOAL_REMINDER`: Daily reminder with quick entry actions
- `GOAL_URGENT`: Time-sensitive alert for imminent derailment

### Watch App Views

#### GoalListView
- Main entry point
- Displays goals sorted by `losedate` (most urgent first)
- Shows urgency indicator, time remaining, and baremin
- Pull-to-refresh support

#### GoalDetailView
- Goal header with urgency badge and stats
- Quick entry buttons (frequent values or defaults: 1, 5, 10)
- Recent entries list
- Reminder settings access

#### DataEntryView
- Custom number pad for value entry
- Optional comment field
- Submit button with loading state

#### SettingsView
- API key input (SecureField)
- Username input
- Test connection button
- Help text with Beeminder link

### Complications (WidgetKit)

Located in `beeWatch Watch App/Complications/`.

**Supported Families:**
- `accessoryCircular`: Shows safebuf days count
- `accessoryRectangular`: Goal title, time remaining, baremin
- `accessoryCorner`: Safebuf days with curved text
- `accessoryInline`: Goal title and time remaining

## Build System

### XcodeGen

Project uses [XcodeGen](https://github.com/yonaskolb/XcodeGen) for project file generation.

```bash
# Install
brew install xcodegen

# Generate .xcodeproj
xcodegen generate
```

**Configuration:** `project.yml`

### Targets

| Target | Platform | Type | Bundle ID |
|--------|----------|------|-----------|
| beeWatch | iOS | Application | com.beewatch.ios |
| beeWatch Watch App | watchOS | Application | com.beewatch.ios.watchkitapp |
| beeWatch Watch App Complications | watchOS | Widget Extension | com.beewatch.ios.watchkitapp.complications |

### Build Commands

```bash
# Build for simulator
xcodebuild -project beeWatch.xcodeproj \
  -scheme "beeWatch Watch App" \
  -destination "platform=watchOS Simulator,name=Apple Watch Series 10 (46mm)" \
  build

# Build for device (requires signing)
xcodebuild -project beeWatch.xcodeproj \
  -scheme "beeWatch Watch App" \
  -destination "generic/platform=watchOS" \
  build

# Clean build
xcodebuild clean build ...
```

### Simulator Commands

```bash
# Boot simulator
xcrun simctl boot "Apple Watch Series 10 (46mm)"

# Install app
xcrun simctl install booted path/to/app.app

# Launch app
xcrun simctl launch booted com.beewatch.ios.watchkitapp

# Uninstall
xcrun simctl uninstall booted com.beewatch.ios.watchkitapp
```

## Data Flow

### Goal Refresh Flow

```
User pulls to refresh
        │
        ▼
GoalListView.refreshable
        │
        ▼
DataStore.refreshGoals()
        │
        ▼
BeeminderAPI.fetchGoals()
        │
        ▼
HTTP GET /users/me/goals.json
        │
        ▼
Parse [GoalResponse]
        │
        ▼
Convert to [Goal], sort by losedate
        │
        ▼
Update DataStore.goals
        │
        ▼
SwiftUI updates GoalListView
```

### Data Entry Flow

```
User taps quick entry button
        │
        ▼
QuickEntryButton.submitValue()
        │
        ▼
DataStore.submitDatapoint()
        │
        ▼
BeeminderAPI.createDatapoint()
        │
        ▼
HTTP POST /users/me/goals/{slug}/datapoints.json
        │
        ▼
Record frequent value
        │
        ▼
Refresh goals list
        │
        ▼
Show success checkmark
```

## Security Considerations

1. **API Key Storage**: Stored in iOS Keychain, not UserDefaults
2. **HTTPS Only**: All API calls use HTTPS
3. **No Credential Logging**: API key never logged or displayed
4. **Request ID**: Each datapoint submission includes unique `requestid` for idempotency
